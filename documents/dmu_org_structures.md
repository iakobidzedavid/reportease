# DMU Organizational Structure Templates
## Reference Guide for Target Digital Marketing Agencies (10-50 employees)

---

## Typical Org Structures by Agency Size

### TEMPLATE 1: Small Agency (10-15 employees)

```
Owner / Managing Director
│
├─ Head of Marketing (1)
│  │
│  ├─ Marketing Manager / Account Manager (1-2)
│  │  └─ Roles: Client reporting, campaign execution, ad management
│  │      Pain Point: Drowning in weekly reporting (5-8 hrs/week)
│  │
│  └─ Paid Media Specialist (1)
│     └─ Role: Ad account management, performance optimization
│        Uses: Google Ads, Facebook Ads Manager directly
│
├─ Project Manager / Operations (0.5 FTE)
│  └─ Role: Tool management, some data work
│     Pain Point: Managing spreadsheets, manual integrations
│
└─ Finance / Admin (1)
   └─ Role: Bookkeeping, invoicing
      May review SaaS budget if >$500/month
```

**DMU at Small Agency:**
- **Economic Buyer:** Owner or Head of Marketing (makes decision <$2K/month independently)
- **End User:** Head of Marketing + Marketing Manager (same person often doing reporting)
- **Influencer:** Project Manager (if present) or IT contractor
- **Budget Gate:** Owner approval if >$500/month annual spend
- **Sales Cycle:** 4-6 weeks (faster; fewer stakeholders)
- **Best Entry Point:** Direct outreach to Owner + Head of Marketing together

### TEMPLATE 2: Growing Agency (20-30 employees)

```
Owner / CEO
│
├─ Chief Marketing Officer (1)
│  │
│  ├─ Client Success Manager / Head of Client Services (1)
│  │  │
│  │  ├─ Marketing Manager / Account Manager (2-3)
│  │  │  └─ Roles: Weekly client reporting, campaign management, ad execution
│  │  │     Pain: 6-8 hrs/week per person on reporting tasks
│  │  │
│  │  └─ Junior Marketer / Marketing Coordinator (1)
│  │     └─ Role: Report assembly, data entry, client communication
│  │
│  └─ Analytics Manager / SEO Manager (1)
│     └─ Role: Campaign optimization, data analysis, strategy
│        Uses: Google Analytics, data aggregation tools
│
├─ Operations Manager (1)
│  │
│  └─ Role: Tool stack management, data integration, workflow automation
│     Pain: Managing 8-10 different tools, manual data syncing
│     Concern: Integration complexity, implementation timeline
│
└─ Finance Manager (0.5-1)
   └─ Role: Budgeting, vendor management, financial reporting
      Approves: SaaS spend >$1K/month
```

**DMU at Growing Agency:**
- **Economic Buyer:** CMO (primary decision-maker; has budget authority up to $3K-5K/month)
- **End User:** Marketing Managers (2-3 people); most time spent on reporting
- **Influencer:** Operations Manager (technical integration, implementation concerns)
- **Budget Gate:** Finance Manager reviews contracts; CMO approves final decision
- **Sales Cycle:** 6-8 weeks (moderate complexity; more stakeholders)
- **Best Entry Point:** CMO for business case; introduce Ops Manager in Week 2

```
   │
   ├─ Head of Client Services (1)
   │  │

   │  │  └─ Role: Client relationships, reporting delivery, strategy
   │  │
   │  └─ Project Manager (1)
   │     └─ Role: Client project coordination
   │
   ├─ Head of Digital Strategy / Analytics (1)
   │  │
   │  ├─ Analytics Manager (1)
   │  │  └─ Role: Campaign analysis, insights, reporting requirements
   │  │
   │  ├─ Performance Manager (1)
   │  │  └─ Role: Paid media optimization, performance tracking
   │  │
   │  └─ SEO Manager (1)
   │     └─ Role: SEO campaigns, organic performance tracking
   │
   └─ Creative Director (1)
      └─ Role: Strategy, creative execution
```

**DMU at Established Agency:**
- **Economic Buyer:** CMO (primary decision authority; owns P&L for marketing function)
- **End User:** Account Managers (3-4 people); highest volume of reporting work
- **Influencer:** Marketing Operations Manager (integration authority; implementation gatekeeper)
- **Budget Gate:** CFO or Finance Manager (contract review, budget approval >$5K/month)
- **Sales Cycle:** 8-10 weeks (complex; multiple stakeholders; formal approval processes)
- **Best Entry Point:** CMO for strategic pitch; fast-track Ops Manager for technical validation

## DMU Decision Authority Map

### Who Decides What at Different Agency Sizes

| Decision | Small Agency (10-15) | Growing (20-30) | Established (35-50) |
|----------|---------------------|-----------------|-------------------|
| **Tool selection** | Owner + Head of Marketing | CMO + Ops Manager | CMO + Ops Manager + CFO |
| **Budget approval** | Owner (any SaaS) | CMO ($1-3K/mo); Finance (>$3K) | CMO + CFO (process) |
| **Implementation** | Owner or Head of Marketing | Ops Manager | Ops Manager (with CMO oversight) |
| **User training** | Head of Marketing | Marketing Manager + Ops Manager | Marketing Manager + Ops Manager |
| **Contract terms** | Owner negotiates | CMO + Finance negotiate | Finance Manager negotiates |
| **Contract signature** | Owner signs | CMO or Finance Manager | Finance Manager or CFO |

## Persona Discovery Questions by Role

### Questions to Ask CMO in Discovery Call

**Current State:**
- "How many hours per week is your team spending on client reporting?"
- "How many people are involved in the reporting process?"
- "What tools are you currently using to compile reports?"
- "Where do errors most commonly occur in your current process?"

**Impact & Priorities:**
- "What's your biggest frustration with the current reporting workflow?"
- "Has reporting workload affected team morale or turnover?"
- "If you could reclaim 6+ hours/week per team member, how would they spend that time?"
- "What would a 'better reporting solution' look like to you?"

**Business Case:**
- "What's your average cost per team member per hour?" (to calculate ROI)
- "How many client relationships could you add if you had 6 extra hours/week?"
- "What's your primary goal for 2026 — growth or profitability?"
- "Do you have budget available for a new SaaS tool in Q2/Q3?"

**Timeline & Decision:**
- "What's your decision timeline?"
- "Who else needs to be involved in the tool selection?"
- "What would you need to see to move forward?"

### Questions to Ask Operations Manager in Technical Review

**Current Infrastructure:**
- "What's your current marketing tech stack? (List all tools)"
- "Which tools are your primary data sources?" (GA, HubSpot, Ad platforms, etc.)
- "How are you currently syncing data between platforms?"
- "Where do you see the biggest data quality issues?"

**Integration Concerns:**
- "How long would implementation typically take for a new tool in your stack?"
- "What's the approval process for new tools?"
- "Do you have any SOC2 or security requirements we need to meet?"
- "What's your preference: API integration vs. UI-based setup?"

**Support & Maintenance:**
- "Who would own the ongoing integration maintenance?"
- "What level of support do you typically need from vendors?"
- "How do you handle API updates from third-party platforms?"
- "What metrics do you track for reporting accuracy?"

### Questions to Ask Marketing Manager in User Interview (POC Phase)

**Current Workflow:**
- "Walk me through your typical weekly reporting process from start to finish."
- "How long does each step take? (data collection, validation, formatting, delivery)"
- "What's the most frustrating part?"
- "How many clients do you report on? How often?"

**Desired Solution:**
- "What would an ideal reporting tool look like for you?"
- "What's the minimum time you'd need to achieve on a weekly report?"
- "What data do you need most visibility into?"
- "Do you need to customize reports for different clients?"

**Adoption:**
- "What would make you excited to use a new tool?"
- "What could prevent you from adopting it?"
- "Would you be willing to be a beta tester?"

## Buying Committee Structure by Size & Type

```
Primary Decision-Maker (CMO)
├─ Technical Evaluator (Ops Manager)
├─ Budget Gatekeeper (CFO / Finance Manager)

```

```
Primary Decision-Maker (Owner or Head of Marketing)
└─ Budget Authority (Owner)
```

## Sales Conversation Sequence by Agency Size

### Small Agency (10-15 people) — 4-6 Week Cycle

| Week | Who | Conversation | CTA |
|------|-----|--------------|-----|
| Week 1 | Owner + Head of Marketing | Business case: time savings, ROI, team morale | Schedule demo + POC |
| Week 2-3 | Same | Live demo on their data; discuss implementation | Decide to proceed |
| Week 3-4 | Owner | Pricing discussion, contract review | Sign contract |
| Week 4-6 | Owner + Head of Marketing | Implementation, training, go-live | Launch |

### Growing Agency (20-30 people) — 6-8 Week Cycle

| Week | Who | Conversation | CTA |
|------|-----|--------------|-----|
| Week 1 | CMO | Discovery: pain point validation, business case | Introduce Ops Manager |
| Week 2 | Ops Manager | Technical review: integration, timeline, data accuracy | Technical sign-off |
| Week 3-4 | CMO + Ops Manager | POC: real data, time savings validation | Decision checkpoint |
| Week 4-5 | CMO + Finance Manager | ROI review, pricing, contract negotiation | Budget approval |
| Week 5-6 | CFO or Finance Manager | Contract review and signature | Contract signed |
| Week 6-8 | Ops Manager + Marketing Team | Implementation, integration, training, go-live | Launch |

### Established Agency (35-50 people) — 8-10 Week Cycle

| Week | Who | Conversation | CTA |
|------|-----|--------------|-----|
| Week 1 | CMO | Discovery call: pain point, strategic fit | Recommend Ops Manager review |
| Week 2-3 | Ops Manager + SE | Technical deep-dive: integration architecture, timeline | Ops sign-off or concerns |
| Week 3-4 | CMO + Ops Manager | POC planning and setup | POC kickoff |
| Week 4-6 | Ops Manager + Marketing Manager | POC execution: real data, workflows, time tracking | POC review meeting |
| Week 6-7 | CMO + CFO + Finance Manager | ROI modeling, pricing, contract terms | Budget approval |
| Week 7-8 | CFO + Finance | Legal review, final negotiation | Contract signed |
| Week 8-10 | Ops Manager + Marketing Team | Implementation, integration testing, training, go-live | Launch |

## DMU Conflict Resolution Strategies

### Scenario 1: CMO Wants Quick Implementation; Ops Manager Says "Too Fast"

**Root Cause:** CMO focused on time-to-value; Ops Manager focused on risk mitigation and integration quality

**Resolution:**
- **Propose phased rollout:** "Week 1: Set up integrations and run on staging. Week 2: Limited production testing with 1 client. Week 3: Full rollout."
- **Address Ops concerns directly:** Outline quality assurance steps, data validation checks, rollback plan
- **Give Ops Manager ownership:** "You're responsible for quality; tell us what timeline gives you confidence"
- **Outcome:** Usually can compress concerns from 2-3 weeks to 1-2 weeks with proper planning

### Scenario 2: Finance Manager Blocks Budget; CFO & CMO Disagree on ROI

**Root Cause:** Finance Manager conservative on cost; CMO optimistic on savings; different ROI expectations

**Resolution:**
- **Separate tool cost from value creation:** Position as "investment in team productivity" not just "software subscription"
- **Provide risk-adjusted ROI model:** Show conservative scenario (3 hrs/week saved) vs. aggressive (6 hrs/week saved); Finance usually accepts conservative case
- **Offer pilot program:** "Start with 2-3 users for 3 months at $199/month; prove the business case before full rollout"
- **Get CFO and CMO aligned before Finance:** If executives agree on value, Finance usually doesn't block
- **Outcome:** Pilot approach almost always breaks through; ~80% convert after successful POC

### Scenario 3: Ops Manager Concerned About Data Accuracy/Integration Stability

**Root Cause:** Ops Manager responsible for data integrity; doesn't want to be blamed if numbers are wrong

**Resolution:**
- **Show validation/accuracy mechanisms:** Data reconciliation reports, automated alerts for anomalies, audit logs
- **Propose automated testing:** Set up monthly reconciliation between platform source data vs. ReportEase data
- **Offer ongoing support:** Dedicated support contact for integration issues; SLA on issue resolution
- **Pilot with non-critical client:** Start with a lower-value client (lower risk) to test accuracy before rolling to key accounts
- **Outcome:** Ops Manager becomes champion once trust in data accuracy is proven; usually takes 2-4 weeks

### Scenario 4: Multiple Stakeholders; Unclear Who Makes Final Decision

**Root Cause:** Agencies with unclear org structures; decision authority not explicit

**Resolution:**
- **Ask directly:** "Who typically makes the final decision on tools in this price range?"
- **Validate with CMO:** In meeting with CMO, ask "Do you have budget authority for this, or does [Finance/Owner] sign off?"
- **Get written confirmation:** Once decision-maker identified, get email confirmation of authority
- **Focus energy on decision-maker:** Don't spread efforts across uncertain stakeholders
- **Outcome:** Clear authority usually saves 1-2 weeks of confusion

## Red Flags in DMU Analysis (Deal Risk Indicators)

| Red Flag | Interpretation | Mitigation |
|----------|---|---|
| "Let me get back to you" (after 3+ days delay) | Low priority; not urgent pain | Refocus on CMO urgency; quantify cost of inaction |
| "We'll need to talk to [vague person]" | Unclear decision authority | Ask directly: "Who has final approval?" |
| "We're happy with our current solution" | Not ready to buy; low pain | Ask: "What would need to change for you to reconsider?" |
| "Our IT department has to approve" | External gatekeepers; longer cycle | Loop IT early; address security/compliance concerns upfront |
| No Ops Manager in conversation (should exist) | Technical concerns may emerge late | Request Ops Manager involvement before POC |
| Finance Manager absent from pricing discussion | Budget authority unclear; risk of late rejection | Get Finance Manager involved by Week 5 of cycle |
| "We need to evaluate 5 other tools" | High evaluation burden; extended timeline | Position unique differentiator; compress alternatives |

## Follow-Up Activities After DMU Analysis

### For Sales Team:

1. **Develop role-based sales decks:**
   - CMO deck: ROI, time savings, team productivity, client impact
   - Ops Manager deck: Integration architecture, implementation timeline, data accuracy
   - CFO deck: Financial model, payback period, vendor stability

2. **Create battle cards for common objections:**
   - By role (CMO concerns vs. Ops Manager concerns vs. Finance concerns)
   - By company size (small, growing, established)

3. **Build discovery call scripts:**
   - Initial CMO discovery (pain validation, ROI exploration)
   - Ops Manager technical review (integration assessment)
   - Marketing Manager user interview (workflow validation)

4. **Design POC process:**
   - Timeline and resource requirements
   - Success metrics and data collection
   - Handoff to implementation team

### For Product Team:

1. **Validate integration support:**
   - Confirm pre-built integrations work smoothly with Google Analytics, HubSpot, ad platforms
   - Stress-test multi-source data syncing
   - Document API implementation timeline (target: <1 week setup)

2. **Build data accuracy validation:**
   - Create reconciliation reports comparing source data to ReportEase output
   - Implement automated anomaly detection
   - Document error handling and alerting

3. **Design implementation playbook:**
   - Step-by-step integration checklist
   - Typical timeline by tech stack complexity
   - Customer success templates for training

4. **Develop ROI calculator tool:**
   - Interactive web tool CMO can use in demo
   - Input: team size, current reporting hours, hourly cost
   - Output: annual savings, payback period, 3-year value
